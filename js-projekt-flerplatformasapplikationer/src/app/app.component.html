Why Angular?


Mer eller mindre alla ramverk har sina styrkor och svagheter, och olika utvecklare har naturligtvis sina egna åsikter och preferenser för dessa. Vilket ramverk som är rätt för ett projekt kan bero på utvecklingsmetoder, utvecklarnas kompetens eller organisationen. Anledningarna kan vara många. Men beroende på vad för projekt som ska utvecklas, kan det ena ramverket vara mer lämpligt än den andra.

En stor fördel är att Angular är det mest välutvecklade ramverket, och har ett ordentligt stöd i termer av community, användarbidrag och är ett komplett paket med allt som behövs. Inlärningskurvan för Angular kan vara väldigt brant, och de nya koncepten i Angular kan vara avskräckande för förstagångsanvändaren som inte har så mycket erfarenhet. Men för de som lär sig Angular, lär sig också god praxis från början för att programmera för större företag.

Jämfört med andra ramverk som React och Vue, så är Angular ett komplett ramverk. Angular innehåller exempelvis routing system, HTTP requests, moduler och mycket annat. Med React hade det behövts flertal tredjepartsbibliotek för att få tillgång till minst lika mycket funktioner. React är mer som ett bibliotek än ett ramverk. När det kommer till Vue så är ramverket relativt nytt, och långt ifrån lika utvecklat som Angular, i termer av support och uppdateringar.

Slutligen, varför vi som grupp valde Angular är för att det kändes som ett tryggt val som ett fullfjädrat ramverk med ett stort community. Men också för att Angular är populärt och beskrivs som ett standardval vid apputveckling inom industrin.
